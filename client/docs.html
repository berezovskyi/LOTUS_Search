III. Hyperparameters for specific matching functions
slop13can be specified for the 'phrase' matching, it is the edit distance on token level allowed. higher slop adds flexibility of matching, though it has increased computational cost
minmatch70%15%can be specified for the 'terms' matching, to indicate the percentage of query tokens that have to be matched. higher percentage required means more strict matching. 100% acts the same as a terms algorithm with 'AND' logical operator.
cutoff0.0010.005can be specified for the 'conjunct' matching, it is a cutoff value to get rid of commonly occurring terms. If a term occurs in bigger part of the documents than the cutoff value, then it is ignored.
fuzziness13can be specified for the 'fuzzyconjunct' matching, it is the edit distance on character level allowed. higher fuzziness adds flexibility of matching, though it has increased computational cost
